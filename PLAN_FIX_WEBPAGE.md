# Plan: Fix modeling-conf-deadlines webpage

## Verified issues

### 1. Checkboxes do not work (root cause: `sub` is a string)
- In the YAML, `sub: SE` is a **string**. In Liquid, `{% for sub in conf.sub %}` over a string iterates **by character**, so `sub` becomes `"S"` then `"E"`.
- So each item gets classes **S-conf** and **E-conf**, not **SE-conf**.
- The filter checkboxes (Programming Languages, Software Engineering, Databases) target **.SE-conf**, **.PL-conf**, **.DB-conf**. No element has **.SE-conf**, so show/hide logic never matches and checkboxes appear to do nothing.
- The JS also does `sub2name["{{sub}}"]` → `sub2name["S"]`, which is **undefined** (types only have SE, PL, DB). So **`undefined.toLocaleLowerCase()` throws** and the rest of the script (deadline display, countdown, etc.) never runs. That is why **no deadline is shown**.

**Fix:** In the sync script, always output `sub` as an **array**, e.g. `sub: [SE]`, so that `{% for sub in conf.sub %}` runs once with `sub = "SE"`, producing class **SE-conf** and valid `sub2name["SE"]`.

### 2. Journals appear under Conferences and Journals section is wrong
- In `_data/conferences.yml`, SoSyM and other journals have **type: conference** instead of **type: journal**.
- So the layout’s `{% if conf.type != "journal" %}` lets them through into the Conferences section, and `{% if conf.type == "journal" %}` in the Journals section finds nothing (or fewer entries).
- Cause: existing entries loaded from file (from a run before type was set correctly) stay in `merged`; `ensure_canonical` only **adds** when link is missing, so it does not **update** type for already-present journals.

**Fix:** In `ensure_canonical`, after building the list, **normalize type for journals**: for every entry whose `link` is in the set of canonical journal links, set `type` to `"journal"`. Also ensure `canonical_entry(..., is_journal=True)` sets `type: "journal"` (already does; keep it).

### 3. Empty deadline text
- As above: JS throws on `sub2name["S"]` so it never reaches the branch that sets `.timer` and `.deadline-time` to `"TBA"` or the countdown. Fixing (1) fixes this.

### 4. Meta display (date/place) for TBA entries
- For TBA entries, `conf.date` and `conf.place` can be empty, so we get “. .” in the layout. Prefer hiding the date/place block when empty, or showing “TBA” / “—” so the page doesn’t look broken.
